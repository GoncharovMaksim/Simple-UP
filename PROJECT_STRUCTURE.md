# Структура проекта Simple UP

## Обзор

Проект содержит верстку 3 экранов по макету Figma и полную интеграцию с WordPress через шорткоды.

## Структура файлов

```
Simple-UP/
│
├── index.html                      # HTML верстка для предпросмотра
├── css/
│   └── style.css                  # Стили для статической верстки
│
├── wordpress-theme/                # WordPress тема
│   ├── style.css                  # Заголовок темы (метаданные)
│   ├── functions.php              # Функции темы (подключение стилей и шорткодов)
│   ├── index.php                  # Основной шаблон страницы
│   ├── header.php                 # Шапка сайта
│   ├── footer.php                 # Подвал сайта
│   ├── shortcodes-examples.txt    # Примеры использования шорткодов
│   │
│   ├── inc/                       # Вспомогательные файлы
│   │   └── shortcodes.php        # Реализация шорткодов
│   │
│   └── assets/                    # Ресурсы темы
│       └── css/
│           └── style.css         # Стили темы WordPress
│
├── docker-compose.yml             # Конфигурация Docker для WordPress
├── .gitignore                     # Игнорируемые файлы Git
│
├── README.md                      # Основная документация
├── INSTALL.md                     # Подробная инструкция по установке
├── QUICKSTART.md                  # Быстрый старт
└── PROJECT_STRUCTURE.md           # Этот файл

```

## Описание компонентов

### 1. Статическая верстка

#### `index.html`
HTML файл с версткой трех экранов:
- БЛОК 1: Header + Hero Section
- БЛОК 6-7: Features Section
- БЛОК 8: Process Section

#### `css/style.css`
Стили для статической верстки с темной темой и фиолетовыми акцентами.

### 2. WordPress тема

#### `wordpress-theme/style.css`
Заголовок темы с метаданными для WordPress.

#### `wordpress-theme/functions.php`
- Подключение стилей темы
- Регистрация шорткодов
- Регистрация меню

#### `wordpress-theme/inc/shortcodes.php`
Реализация трех шорткодов:
1. `[simple_up_hero]` - Hero секция с header
2. `[simple_up_features]` - Секция с возможностями
3. `[simple_up_process]` - Секция с процессом работы

#### `wordpress-theme/assets/css/style.css`
Стили темы WordPress (идентичны статической верстке).

### 3. Docker конфигурация

#### `docker-compose.yml`
Конфигурация для запуска WordPress локально:
- WordPress контейнер (порт 8080)
- MySQL база данных
- phpMyAdmin (порт 8081)

## Верстка экранов

### БЛОК 1: Header + Hero Section
- Навигационное меню с логотипом
- Главный заголовок
- Описание
- Кнопка призыва к действию
- Градиентный фон

### БЛОК 6-7: Features Section
- Заголовок секции
- Сетка из 4 карточек возможностей
- Иконки, заголовки, описания
- Эффекты при наведении

### БЛОК 8: Process Section
- Заголовок секции
- Два столбца:
  - Левая колонка: элементы процесса с иконками
  - Правая колонка: пронумерованные шаги

## Шорткоды

### `[simple_up_hero]`
Параметры:
- `title` - Заголовок (по умолчанию: "Добро пожаловать в Simple UP")
- `description` - Описание
- `button_text` - Текст кнопки (по умолчанию: "Начать работу")
- `button_link` - Ссылка кнопки (по умолчанию: "#")

### `[simple_up_features]`
Параметры:
- `title` - Заголовок секции (по умолчанию: "Наши возможности")
- `columns` - Количество колонок (по умолчанию: "4")

### `[simple_up_process]`
Параметры:
- `title` - Заголовок секции (по умолчанию: "Как мы работаем")

## Разработка

### Локальная разработка

1. Откройте `index.html` в браузере для предпросмотра верстки
2. Для WordPress используйте Docker или локальный сервер

### Изменение стилей

Все стили находятся в `wordpress-theme/assets/css/style.css`.
Изменения применяются автоматически после обновления файла.

### Добавление новых шорткодов

1. Добавьте функцию в `wordpress-theme/inc/shortcodes.php`
2. Зарегистрируйте шорткод через `add_shortcode()`
3. Добавьте стили в `wordpress-theme/assets/css/style.css`
