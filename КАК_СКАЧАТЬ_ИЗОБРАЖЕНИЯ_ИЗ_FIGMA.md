# Как скачать разные изображения людей из Figma

## Проблема
В макете Figma есть 4 разных изображения людей, которые должны использоваться в разных карточках:
1. **person-1.png** - для карточки "Сайт не приносит клиентов"
2. **person-2.png** - для карточки "SEO без результатов"  
3. **person-3.png** - для карточки "Сомнения и недоверие"
4. **person-4.png** - для блока решения

## Решение: Экспорт изображений из Figma

### Способ 1: Экспорт через интерфейс Figma

1. **Откройте макет в Figma:**
   - Перейдите по ссылке: https://www.figma.com/design/nVuL0m8Sil5xmmW4YViKUj/Untitled--Copy-?node-id=39-1502

2. **Найдите каждое изображение человека:**
   - В левой панели найдите элемент с изображением человека
   - Или кликните на изображение человека в макете

3. **Экспортируйте изображение:**
   - Выделите элемент с изображением
   - В правой панели найдите секцию "Export" (Экспорт)
   - Нажмите кнопку "+" чтобы добавить экспорт
   - Выберите формат: **PNG**
   - Выберите размер (рекомендуется 2x или 3x для качества)
   - Нажмите "Export [название]" для скачивания

4. **Переименуйте файлы:**
   - Сохраните файлы с именами:
     - `person-1.png` - изображение из карточки 1 (Сайт не приносит клиентов)
     - `person-2.png` - изображение из карточки 3 (SEO без результатов)
     - `person-3.png` - изображение из карточки 5 (Сомнения и недоверие)
     - `person-4.png` - изображение из блока решения

### Способ 2: Экспорт через контекстное меню

1. **Кликните правой кнопкой мыши** на изображение человека
2. Выберите **"Copy/Paste as" → "Copy as PNG"** (или "Экспортировать")
3. Сохраните изображение с правильным именем

### Способ 3: Массовый экспорт (если все изображения в одном слое)

1. Выделите все изображения людей (удерживая Shift)
2. В правой панели нажмите "Export"
3. Добавьте экспорт для всех
4. Нажмите "Export" для скачивания всех файлов
5. Переименуйте файлы соответственно

## Размещение файлов

После скачивания изображений:

1. **Для WordPress темы:**
   - Скопируйте файлы в: `wordpress-theme/assets/images/`
   - Файлы должны называться: `person-1.png`, `person-2.png`, `person-3.png`, `person-4.png`

2. **Для статической версии:**
   - Скопируйте файлы в: `images/`
   - Файлы должны называться: `person-1.png`, `person-2.png`, `person-3.png`, `person-4.png`

## Проверка

После размещения файлов проверьте:

1. **В WordPress:**
   - Откройте страницу в браузере
   - Откройте консоль разработчика (F12)
   - Проверьте вкладку "Network" - все изображения должны загружаться без ошибок 404

2. **В статической версии:**
   - Откройте `index.html` в браузере
   - Проверьте, что все изображения отображаются корректно

## Текущее состояние

Сейчас в проекте созданы файлы-заглушки:
- `person-1.png`, `person-2.png`, `person-3.png`, `person-4.png`

Эти файлы являются копиями `person-illustration.png` и служат временными заглушками.

**Нужно заменить их на реальные изображения из Figma!**

## Node IDs в Figma (для справки)

Если вы используете Figma API или плагины:
- Карточка 1: node-id для изображения нужно найти в макете
- Карточка 3: node-id для изображения нужно найти в макете
- Карточка 5: node-id для изображения нужно найти в макете
- Блок решения: node-id для изображения нужно найти в макете

## Альтернативный способ: Использование Figma API

Если у вас есть доступ к Figma API:

```bash
# Получить изображение по node-id
curl -H "X-Figma-Token: YOUR_TOKEN" \
  "https://api.figma.com/v1/images/FILE_KEY?ids=node-id&format=png&scale=2" \
  -o person-1.png
```

Но проще использовать экспорт через интерфейс Figma.
